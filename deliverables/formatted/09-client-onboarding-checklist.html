Client Onboarding Checklist

Use this checklist every time you sign a new client. This is directly from the 6-step delivery framework. The onboarding call is one of the most important steps in the entire delivery process — you set expectations and get everything you need to build.

Before the Onboarding Call

  • Payment confirmed — No payment, no work. Period.
  • If paid: first installment received (50% upfront or 100% upfront)
  • If free client: clearly outlined what you're giving them AND what they give you (testimonial + 2 referrals)
  • Contract/agreement in place — Even informal, define: what you're building, what "done" looks like, what they provide in return
  • Onboarding call scheduled — Book a 45-60 minute call

The Onboarding Call Agenda

1. Set Timeline Expectations

  • Tell the client how long the project will take
  • Break the project into milestones:
  • "Week 1: Setup and integrations"
  • "Week 2: Building the core automation"
  • "Week 3: Testing"
  • "Week 4: Final tweaks and handoff"
  • Underpromise by 1-2 weeks. If you think it takes 3 weeks, tell them 4-5 weeks. If you deliver early, you overdeliver.

3. Choose ONE Communication Channel

  • Pick ONE platform and stick to it. Don't email AND text AND Slack AND WhatsApp.
  • Options: WhatsApp (free, easy), Slack (professional, organized), or whatever the client prefers
  • Ask them: "What's the easiest way for us to stay in touch throughout the project?"

4. Get All Software Access

This is critical. Do NOT get off the call without having everything you need.

  • Automation platform access:
  • Have the client create their own account (Make/n8n/Voiceflow)
  • Have them add you as a user/collaborator in their account

  • Walk them through it on the call: "Can you share your screen? Go to Settings → Add User → enter my email"

  • API keys:

  • The client creates and pays for API accounts (OpenAI, Claude, Twilio, etc.)
  • They generate the API key and share it with you

  • The client pays for all API costs

  • Other software access:

  • CRM login (if integrating with their CRM)
  • Email marketing platform access (if relevant)

  • Any other tools they currently use that you'll need to connect

  • Who pays for what:

  • Client pays for: all software subscriptions, API keys, hosting
  • You pay for: nothing (unless you're hosting the automation on your own account for a retainer model)

Exception — Retainer/hosted model:
If you're charging a retainer to maintain the system (e.g., a voice agent), you may host it on your own account. This creates dependency — the client stays because you manage everything. But they still pay for APIs (OpenAI, Twilio, etc.).

Software access checklist:
- Automation platform account created and you've been added
- API keys generated and shared with you
- CRM/other tool access provided
- All software costs on the client's accounts

After the Onboarding Call

  • Send a confirmation message summarizing what was discussed:

"Hey [client name], great call today! Just to recap:

  • Project: [what you're building]
  • Timeline: [X weeks], with the first milestone ([milestone 1]) by [date]
  • Communication: We'll use [WhatsApp/Slack] for all project communication
  • I'll send you weekly updates every [day of week]
  • Next step: I'll start working on [first milestone] and update you by [date]

Excited to get started!"

  • Add the client to your KPI tracker (Client Tracker tab)
  • Set up your weekly update reminder (pick a day — send updates the same day every week)
  • Start working on Milestone 1
